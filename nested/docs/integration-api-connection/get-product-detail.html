Getting the Product#

This feature enables you to create a new transaction using the minimum data required and get quotation URL as the result.

Endpoint:

{base-url}/api/v1/products/slug/{product_slug}/?gender={gender}&dob={dob}&is_cashless={is_cashless}

Payload:

KeyRuleTypeDescriptionExample
product_slugrequiredstringSlug of the insurance productsuper-care-protection
is_cashlessoptionalbooleanSpecify cashless claim producttrue / false (default: true if null)
genderoptionalstringSpecify male or female for gender specific productmale / female
doboptionalstringSpecify date of birth for premium calculation2001-12-31 - yyyy-mm-dd
